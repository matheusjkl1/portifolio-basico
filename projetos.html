Blissful Finance 💰✨

Plataforma moderna de controle financeiro pessoal enriquecida com Inteligência Artificial.
Next.js 15 LangGraph Prisma PostgreSQL

Principais Entregas e Impactos

O Blissful Finance foi concebido para simplificar a gestão financeira. O maior diferencial técnico está no uso de IA para entrada de dados em linguagem natural e na consistência das regras contábeis fundamentais.

  • Assistente Conversacional NLP (Quick Add): Integração de agente inteligente via LangGraph que extrai transações de falas do usuário (ex: "R$ 20 em comida no Nubank"), deduzindo os parâmetros e realizando perguntas adicionais caso falte informação.
  • Moderação e Economia de Tokens: Criação de guardrail de intenção na camada de IA que bloqueia discussões fora de contexto (ex: dúvidas de código), otimizando em larga escala o uso de tokens da API.
  • Divisão Inteligente de Despesas: Algoritmo para cálculo de splits de gastos. O criador mantém a transação integral no cartão de crédito, enquanto o split é refletido como receita esperada no dashboard, mantendo a integridade contábil.
  • Painel Analytics & BI: Refatoração do módulo de previsões, permitindo agrupamento de despesas por períodos dinâmicos e renderização de dashboards interativos de receitas vs. despesas.

Decisões de Engenharia & Stack

Linguagens: TypeScript, SQL Frontend: React 19, Next.js 15 (App Router), Tailwind CSS v4, shadcn/ui, Recharts Backend: Next.js API Routes, Supabase Auth Banco de Dados: PostgreSQL, Prisma ORM Inteligência Artificial: LangGraph, LangChain, OpenRouter API Qualidade & CI: Jest (Unitários/Integração), Playwright (E2E), GitHub Actions

Boas Práticas Aplicadas

  • Regra Monetária Estrita: Armazenamento e manipulação de valores financeiros como inteiros representativos de centavos (`Int`), evitando erros de arredondamento em pontos flutuantes.
  • Repository Pattern: Isolamento absoluto da camada de banco de dados (`Prisma`) das rotas de negócio, propiciando mockabilidade total nos testes Jest.
  • React Server Components (RSC): Renderização prioritária no servidor para buscar dados diretamente do banco, diminuindo latências de ida e volta ao cliente.

Fitness AI Platform 🚀

Plataforma fitness inteligente baseada em IA contextual (PWA) e streaming de vídeo de alta performance.
Next.js 15 pgvector Edge Functions HLS Streaming

Principais Entregas e Impactos

Plataforma fitness (Mobile-First / PWA) focada em resolver as alucinações das IAs comuns na recomendação de treinos físicos. Combina busca semântica em banco vetorial de catálogo técnico real e streaming HLS.

  • Busca Híbrida Semântica: Desenvolvimento de funções RPC no PostgreSQL que cruzam buscas vetoriais (`pgvector`) com parâmetros relacionais de segurança (restrições médicas do usuário, equipamentos e nível físico).
  • Pipeline Resiliente de Embeddings: Worker TypeScript que enriquece semanticamente os exercícios e orquestra a geração de embeddings (1536 dimensões via modelo text-embedding-3-small) com controle de rate limiting e retentativas.
  • Streaming de Vídeo HLS: Conversão automatizada de guias de exercícios em blocos fragmentados HLS (`.m3u8` e segmentos `.ts`) armazenados no Supabase Storage, resultando em carregamento instantâneo do reprodutor sem travar redes móveis.
  • Autenticação Anônima: Barreira zero no onboarding utilizando sessões anônimas do Supabase, convertendo depois o visitante em usuário registrado sem quebrar seu histórico de treinos.

Decisões de Engenharia & Stack

Linguagens: TypeScript, SQL Frontend: Next.js 15, React 19, Tailwind CSS v4, PWA (Mobile-First) Backend & BD: Supabase (Auth, Storage), PostgreSQL com pgvector extension Edge & Workers: Deno Edge Functions, Node.js (TS Ingestion Workers) Mídia: HTTP Live Streaming (HLS) Monorepo: Yarn 4 (Plug'n'Play), Turborepo

Lições Técnicas & Conquistas

  • Mitigação de Alucinação: A pesquisa comprovou que a junção de busca semântica indexada localmente em vez de depender apenas de contexto injetado direto no LLM aumentou a segurança em 100% nas fichas de treino.
  • Eficiência de Custo: A persistência e busca vetorial indexada no PostgreSQL barateou a conta de API em mais de 70% comparado a pipelines de prompt engineering convencionais.
  • Yarn PnP & Turborepo: Redução drástica do espaço de disco local e tempo de build compartilhando tipagens e pacotes comuns no monorepo.
